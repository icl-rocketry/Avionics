\doxysubparagraph*{Enhancements}

The library now does the following -\/ if the flash memory chip is compatible with the SFDP standard\+:
\begin{DoxyItemize}
\item Identifies the flash memory chip using the Serial Flash Discoverable Parameters standard. (JESD216B)
\item Identifies the following opcodes from the SFDP tables and uses them to read/write data\+:
\begin{DoxyItemize}
\item {\ttfamily erase\+Sector}
\item {\ttfamily erase\+Block32K}
\item {\ttfamily erase\+Block64K}
\end{DoxyItemize}
\item Reads timing information from SFDP to accurately time the following operations\+:
\begin{DoxyItemize}
\item {\ttfamily erase\+Sector}
\item {\ttfamily erase\+Block32K}
\item {\ttfamily erase\+Block64K}
\end{DoxyItemize}
\item Fixes \#125 in commit 10c2b86. The library automatically uses the alternate chip erase code (0x\+C7) as required (only on fully supported chips)
\end{DoxyItemize}

\doxysubparagraph*{Enhancements\+:}


\begin{DoxyItemize}
\item A new function -\/ {\ttfamily flash.\+erase\+Section(address, size)} -\/ has been introduced in this version. When a user requires a large and variable (between writes) amount of data to be written to the flash memory on the fly and to have the correct amount of space erased to fit the data, this function will automatically calculate and erase the right amount of space to fit the data. Please note that if the the amount of data being written is consistently the same size, the pre-\/existing {\ttfamily flash.\+erase\+Sector()}, {\ttfamily flash.\+erase\+Block32\+K()} and {\ttfamily flash.\+erase\+Block64\+K()} functions will operate a lot faster.
